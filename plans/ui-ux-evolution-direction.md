# UI / UX 规则进化方向

> 状态：`-direction`（初步方向，待讨论对齐）

## 背景

现行 UI 规则体系（`docs/ui-design/`）以**克制、规整、减法**为核心方法论。
这套方法论在初期快速拉齐了质量底线，但也暴露了一个结构性副作用：

**规则系统擅长约束"不要做什么"，但缺少指导"如何把该做的做出质感"。**

这导致 AI 生成的页面和人写的页面，都容易滑向一种相似的"灰安全区"——
结构正确、间距合规、颜色合法，但整体观感是冷的、平的、缺少节奏的。

本文记录问题观察和潜在进化方向，不改动现行规则，不定实施优先级。

## 观察到的具体问题

### 1. Document Block 缺乏首行焦点

当前的 label/value 信息块：所有字段等权重排列，灰色 label + 黑色 value 重复堆叠，
用户看到的是均匀的"表格感"，没有扫读入口。

**方向**：允许 document block 有"主行"——用图标 + strong 文字建立首行焦点，
次要字段缩进横排。这不是装饰，是信息架构里的视觉锚点。

### 2. 步骤流的过渡缺乏连续感

当前步骤切换是硬切——上一步消失、下一步出现，用户感知不到"推进"。
antd `Steps` 太重且不匹配品牌气质，纯文字"步骤 N / 3" 又太冷。

**方向**：轻量自定义 step indicator，图标节点 + 连接线 + 状态色过渡。
已在 invite 流程中验证了基本形态，可作为公共入口流程的通用模式。

### 3. 成功态仪式感不足

当前成功状态用 `Alert type="success"` 或等权重文案，
和错误状态的视觉声量几乎相同。但成功是用户流程的终点奖励，值得更多一点声量。

**方向**：成功终态允许居中布局 + 放大成功图标（40–48px）+ 标题/说明两行分层 +
主操作按钮升为 `size="large"`。不需要动画或 confetti，结构本身就能给出仪式感。

### 4. 按钮缺少方向暗示

当前所有按钮都是纯文字，前进/后退/提交/取消的操作方向完全靠文案传达。
用户扫读时无法快速区分主操作意图。

**方向**：前进类操作加 `RightOutlined` + `iconPosition="end"`，
后退加 `LeftOutlined` 前缀，提交确认加 `CheckOutlined`，
重试类加 `ReloadOutlined`。图标服务于方向感，不是装饰。

### 5. 页面整体缺少呼吸节奏

"先结构后装饰"的原则被执行得过于字面——结构正确后没有任何装饰层，
导致整页都是白+灰+蓝的同一声部。

**方向**：在不违反品牌调性的前提下，允许以下手段增加节奏：

- 浅色 fill 背景块（`colorFillQuaternary`）做信息分区
- Card 加 `shadow-card` 提升层次
- 不同阶段的 section 之间允许 8px 的呼吸间距差异

## 更远的探索方向

### A. 轻量过渡动画

当前规则只定义了 `duration-150` 和 `duration-200` 两档 wrapper 动画，
且限定在 hover/显隐场景。完全缺少页面级过渡和步骤切换过渡。

潜在方向：

- 步骤切换时，内容区做一次 `opacity 0→1 + translateY 8px→0` 的 150ms fade-in
- 成功态出现时，图标做一次 `scale 0.8→1` 的 200ms 弹入
- 这些过渡不依赖第三方库，纯 CSS transition 或 `@starting-style`
- 需要确认对 reduced-motion 偏好的 respect（`prefers-reduced-motion: reduce` 时跳过）

### B. 公共入口页面的品牌氛围层

登录、邀请、重置密码等公共入口页面，当前只有 `bg-bg-layout`（浅灰）+ 白 Card。
品牌存在感仅通过 BrandLockup logo 传达，整体和任何 SaaS 页面没有区别。

潜在方向：

- 页面底部或侧边加一层极轻的品牌纹理（如 Deep Blue 的 2% opacity 几何图案）
- 或用 CSS `radial-gradient` 在页面边角做一个极淡的品牌色光晕
- **约束**：不能干扰正文阅读、不能在深色模式下刺眼、不能影响表单操作区

### C. Canvas / 动态视觉元素

更远期的探索——在特定公共页面（如登录页、邀请成功页）加入轻量 Canvas 动效：

- 极慢速的粒子漂移或几何连线，Deep Blue + Sky Cyan 双色
- 只在 viewport 左/右空白区域绘制，不覆盖功能区
- GPU 消耗极低（<100 粒子，低帧率），移动端自动降级或关闭
- **这是纯氛围层，与功能无关，需要独立评估 ROI**

## 对现行规则的影响

本文提到的方向**不修改**现行 `docs/ui-design/` 下的任何规则文件。

若某个方向经过验证并稳定：

1. 提炼为带约束的规则条款
2. 写入对应的规则文档（如 `tokens.md` 补动画档位、`ux-guidelines.md` 补成功态模式）
3. 从本 plan 中移除或标记 `→ 已落地`

现行规则的核心哲学不变：**克制但不空**。
本文的意图是补充"不空"这一半——让规则系统不只擅长做减法，也能指导如何精确地做加法。

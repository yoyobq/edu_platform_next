# Vercel (Geist Design) 骨架与质感标杆

## 风格关键词

极简克制、极致排版比例（负字间距）、高对比度、光晕替代阴影、丝滑反馈、1px 细线（Shadow-as-border）。

## 为什么选择 Vercel？

在抛弃了 IBM Carbon 这种过于“传统、老旧”的企业级后台规范后，我们需要一个能够代表**现代 SaaS 平台最高标准**的基准标杆。Vercel（及其底层的 Geist Design System）完美契合了我们的需求：它极其干净、结构极其清晰，且在光影处理（Shadow-as-border）和排版比例上堪称教科书。它“先结构后装饰”的理念，与我们 [README.md](../README.md) 中的设计性格高度一致。

⚠️ **警告：本项目禁止完全复刻 Vercel 的“极客黑白”配色与硬编码的色号/阴影。** 我们是面向师生的平台，必须保留品牌温度。本篇文档**仅用于提取其空间骨架、光影分层逻辑与排版哲学**。所有视觉落地必须使用项目中现有的 Tailwind Token 和 Ant Design 主题。

## 核心可吸收点（视觉与交互）

### 1. 骨架与留白体系 (Whitespace & Structure)

- **画廊般的留白（Gallery Emptiness）**：Vercel 极少使用大面积色块划分区域，而是通过完美的间距节奏和文本颜色深浅来拉开信息层次。区块与区块之间依靠留白呼吸，而非生硬的分割线。
- **排版压迫感与空间释放的平衡**：Vercel 喜欢在标题上使用负字间距（Negative tracking，使其看起来像被压缩的代码），但在周围留出巨大的负空间。这种“内紧外松”的张力是其高级感的来源。

### 2. “阴影即边框”哲学 (Shadow-as-border)

Vercel 最著名的设计签名是不使用传统的 CSS `border`，而是使用 **1px 扩散的 Box Shadow**。

- **复合阴影堆叠**：它的卡片不仅有一层表示物理深度的软阴影，还有一层 1px 的硬边框阴影，甚至内部还有一层极细的高亮内环（Inner ring，如 `#fafafa`），这让卡片看起来像是由材质雕刻而成，而非扁平的纸片。
- **在本项目中的应用**：我们的 `tokens.md` 中定义的 `--ant-box-shadow-secondary` 和 `shadow-card` 应当学习这种多层堆叠的质感，拒绝死板的单层黑影。

### 3. 排版角色与字体功能

- **Geist Mono 的点缀**：在高度技术化或需要强调系统状态的地方（如：AI 状态流转、代码标签），使用等宽字体（Monospace）全大写配合紧凑的字间距，能瞬间提升界面的“专业工具感”。
- **严格的字重（Weight）收敛**：放弃 700（Bold），只使用 400（阅读）、500（交互/UI）、600（强调/标题）。通过字号大小和颜色的明暗来区分主次，而不是把字加得特别粗。

### 4. 高级感的微交互

- **焦点环（Focus Ring）**：在获取焦点时，明确、干净、高饱和度的蓝色边框（Outline/Ring），绝不模糊。
- **状态徽章（Pill Badges）**：极小尺寸的药丸状标签（如：部署状态），带有极高对比度的文字与极浅的同色系背景，克制而清晰。

## 🚫 不可照搬点（严格禁止的视觉特征）

1. **极端的负字间距（Extreme Negative Tracking）**：Vercel 标题的负字距（如 -2.4px）仅在其自研的 Geist 英文字体下表现完美。在我们的中文语境（尤其是师生阅读场景）中，**绝对禁止**对中文字体使用剧烈的负字间距，这会严重破坏中文的易读性。
2. **纯粹的“无边框”输入态**：Vercel 的表单设计有时为了极简会去掉输入框边框。在我们的教育后台场景中，为了保证可用性（Accessibility）和降低认知负担，表单控件应当遵循 Ant Design 的默认带边框形态，不宜过度追求“无边界”。
3. **照搬具体色号与渐变**：Vercel 的 Ship Red、Preview Pink 等属于其专属的 Workflow 颜色。我们不能在代码里写死这些色号，必须使用项目内的品牌色和语义色。

## 在本项目中的适用范围

- **高度适用**：
  - 全局骨架、大容器（Wrapper）、卡片区块（Cards）的划分参照物。
  - 数据大盘、仪表盘（Dashboard）的数值卡片排版。
  - 暗黑模式（Dark Mode）下的阴影与边框光影处理标杆。
  - 侧边栏或列表中的状态徽章（Badges）与微小标签。
- **不适用**：
  - 需要强品牌情感连接的 C 端营销页（Vercel 太冷）。
  - 大段落的中文正文阅读区（需遵循标准的中文排版行高）。

## 给 AI 与开发者的落地要求

当你被要求“参考 Vercel 的设计感”时，请遵守以下映射关系：

1. **排版比例**：不要手写字号和负字间距，必须复用 `typography.md` 里的规范，通过 `text-text-primary` 和 `text-text-secondary` 的对比度来体现高级感。
2. **边框与留白**：多用 `rounded-card`、`rounded-block` 配合适度的 `p-6` 或 `p-4` 间距。利用 `shadow-card` 营造“阴影即边框”的质感。
3. **不要自己发明 CSS**：吸收 Vercel 的理念是“用极简的元素搭建复杂的系统”，这意味着你应该**尽可能少地**写自定义 CSS，而是利用现有的 Ant Design 组件和基础的 Tailwind 工具类组合出干净的界面。

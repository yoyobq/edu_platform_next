# Chart Guidelines

本文件定义项目中的图表、指标卡、数据面板与 dashboard 视图的统一判断标准。

它回答的不是“图表长什么样更酷”，而是：

- 什么情况下应该出图
- 出图前必须先明确哪些口径
- 图表如何保证可判断、可复核、可导出、可解释

若本文件与真实业务定义冲突，以业务定义与后端真实口径为准；若与时间语义冲突，以 [../project-convention/time-display-semantics.md](../project-convention/time-display-semantics.md) 为准。

## 适用范围

适用于：

- 趋势图、对比图、预测图、监控图、经营看板
- 页面中的主图表、卡片内小图、指标摘要卡
- 任何带有时间序列、阈值、目标值、预测值、对比值的可视化

不适用于：

- 纯装饰性插图
- 不承载真实业务判断的 marketing 风格示意图

## 核心原则

### 1. 先定义问题，再定义图表

每张图只回答一个主问题。

先写清：

- 这张图要帮助用户判断什么
- 用户看完后要做什么决策

若一张图同时想回答“趋势如何”“结构如何”“是否异常”“和目标差多少”，说明问题拆分不够，应分成多张图或“主图 + 辅助表格/摘要”。

### 2. 先定义口径，再开始绘制

出图前至少要明确：

- 指标名称
- 指标定义
- 单位
- 时间粒度
- 统计周期
- 聚合方式
- 数据来源

口径不清时，不出图。

### 3. 图表首先服务判断效率

图表风格以专业、克制、可读、现代为主。

允许轻微未来感，但不能牺牲：

- 首屏可判读性
- 系列区分能力
- 单位与口径可见性
- 复核与导出能力

## 必须遵守

### 1. 主问题必须唯一

每张图在实现前，先在代码注释、设计稿或需求文档中写一句主问题描述。

示例：

- 查看最近 30 天实际用电量是否偏离预测
- 查看当前企业在本月内哪些日期缺失预测结果
- 查看当前报价生成量是否达到阶段目标

### 2. 指标口径必须显式

至少在图表标题、副标题、说明区或图旁说明中明确：

- 指标含义
- 单位
- 粒度
- 时间窗

禁止把单位、统计周期、口径说明只藏在 tooltip 里。

### 3. 有 current / latest 概念时必须显式展示

如果图表存在“当前值”“最新值”“最近一次结果”的业务意义，必须在页面可见区域展示：

- `最新值 / 当前值`
- 对应时间点

不能要求用户自己把视线移到最右侧再猜哪一个点是当前值。

### 4. 时间窗必须明确，且可切换或可快速定位最近数据

图表必须明确当前展示的是：

- 最近 24 小时
- 最近 7 天
- 最近 30 天
- 本月 / 本季度 / 自定义区间

并且至少满足一项：

- 支持时间窗口切换
- 支持“回到最近时间 / latest”定位

### 5. 系列语义必须稳定

实际值、预测值、对比值、目标值、阈值必须强制视觉区分。

至少通过两种维度区分，不能只靠颜色：

- 颜色
- 线型（实线 / 虚线 / 点线）
- 线宽
- 点形
- 填充方式
- 标签文案

项目内同一类序列应保持稳定语义，不允许今天“绿色=预测”，明天又“绿色=实际”。

### 6. 阈值、目标值、正常区间必须显式画出

只要业务里存在以下概念，就应直接画出来：

- 正常区间
- 告警阈值
- SLA 边界
- 目标值
- 上下限

不要让用户在脑中自己记阈值。

若业务并不存在可靠阈值，不要为了“看起来完整”凭空造一条参考线。

### 7. 图表必须附一句解释

每张图都应附带一句简短说明，解释：

- 指标在看什么
- 应如何阅读
- 当前颜色 / 线型代表什么

它不需要很长，但必须帮助第一次进入页面的人快速建立阅读方式。

### 8. 图表必须可复核

图表旁至少应提供其中几项，完整 dashboard 优先都提供：

- 表格视图
- 导出
- 更新时间
- 数据来源说明
- 当前筛选条件展示

图表不能成为“只能看、不能验”的黑盒。

## 项目补充决策

以下几条建议加入项目正式规范，比原始 checklist 更适合工程落地。

### 1. 缺失值、零值、未生成值必须分开

这是最容易误导判断的一类问题。

必须明确区分：

- `0`：真实值为零
- `null / missing`：没有数据
- `not generated yet`：预测尚未生成
- `failed / invalid`：数据生成失败或不可用

禁止把“无数据”直接画成 `0`。

### 2. current / latest 必须与图表同口径

页面上的“当前值 / 最新值”必须与图表使用同一套：

- 筛选条件
- 时间窗
- 粒度
- 统计口径

不要出现摘要卡用“本月累计”，图里却是“日值趋势”，但页面又不说明两者关系。

### 3. 对比关系必须写清 baseline

若图中包含：

- 环比
- 同比
- 上期对比
- 预测 vs 实际
- 当前 vs 目标

则必须明确：

- 对比对象是谁
- 差值公式是什么
- 单位是绝对值还是百分比

不要只写“较上期 +12%”而不说明上期是什么。

### 4. 系列颜色语义在项目内固定

建议默认语义：

- 实际值：主阅读序列，实线，颜色更实、更重
- 预测值：辅助判断序列，虚线或弱化样式
- 对比值：中性辅助色，避免与实际值抢主声部
- 目标值 / 阈值：边界线样式，带文字标签
- 缺失 / 异常：告警色，但只在需要提醒时使用

具体颜色取值仍以 [colors.md](./colors.md) 为准，但系列角色不能漂移。

### 5. 表格与导出必须和图表同源

若页面提供表格视图或导出：

- 字段口径必须与图表一致
- 时间范围必须一致
- 当前筛选必须一致
- 导出文件应尽量带上时间窗和筛选条件

不要出现“图里看的是 A 口径，表里导出的是 B 口径”。

### 6. 更新时间与来源不应缺席

对于业务判断有时效性的图表，应展示：

- 更新时间
- 数据来源
- 是否为估算 / 预测 / 缓存结果

尤其是预测页、报表页、任务异步生成页，不能让用户误以为数据是实时最终值。

### 7. 首屏不超过可读上限

默认情况下：

- 首屏主图优先只保留 1 个主问题
- 同一张图的主要序列尽量不超过 4 条
- 不默认使用双轴

如果必须增加信息量，优先拆图，不优先叠图。

### 8. tooltip 只做补充，不做托底

tooltip 可以放：

- 更精确的小数值
- 同一点位的多序列详情
- 补充元信息

tooltip 不应承担：

- 基础单位
- 核心口径
- 阈值定义
- 当前筛选说明

## 图表类型选择速查

| 主问题                         | 优先图表类型                  | 说明                                           |
| ------------------------------ | ----------------------------- | ---------------------------------------------- |
| 看趋势怎么变                   | 折线图 / 面积图               | 时间序列默认优先折线；面积只在强调规模感时使用 |
| 看当前值是否达标               | 指标卡 + 阈值线 / Bullet 风格 | 当前值必须显式，目标与边界必须同屏             |
| 看排名或企业间对比             | 条形图                        | 类目多时优先横向条形图                         |
| 看构成比例                     | 堆叠条形图 / 堆叠面积图       | 先确认主问题真的是“构成”而不是“趋势”           |
| 看实际 vs 预测                 | 双序列折线图                  | 实际与预测必须有稳定的线型差异                 |
| 看异常、越界、是否进入危险区间 | 折线图 + 阈值 / 区间带        | 不要只在 tooltip 提示异常                      |
| 看明细与复核                   | 表格                          | 表格不是降级方案，而是图表的复核配套           |

## 评审清单

Code Review 或设计评审时，至少快速过一遍：

1. 这张图只回答一个主问题吗？
2. 指标定义、单位、粒度、时间窗是否页面可见？
3. latest/current 是否显式展示？
4. 实际、预测、对比、阈值是否能一眼区分？
5. 是否错误地把缺失值画成了零值？
6. 是否存在应该画出的目标值或阈值却没有画？
7. 图旁是否有一句解释？
8. 是否能通过表格、导出、更新时间、来源完成复核？
9. 图表与表格 / 导出是否同口径？
10. 首屏是否足够清晰，而不是靠 tooltip 才看得懂？

## 一句话判断

图表不是为了展示“我们有数据”，而是为了让用户更快、更稳地做出判断。

若一张图不能被解释、不能被复核、不能被导出、不能被稳定阅读，它就还不该上线。

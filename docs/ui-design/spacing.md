# Spacing

项目已有间距档位（见 [tokens.md](./tokens.md)）。本文件补的是**节奏规则**：这些档位在什么结构里、按什么优先级使用。不发明新 token，只定使用规则。

## 核心原则

### 1. 先定结构层级，再定间距大小

不要先想"这里用 `gap-4` 还是 `gap-6`"，应先判断它是：

- 页面级区块之间
- 区块内部条目之间
- 行内元素之间
- 容器内边距
- 单个文本的小修正

层级先定，间距才不会漂。

### 2. 默认优先 `gap`，其次 `padding`，最后才是 `margin`

| 场景               | 优先选择 |
| ------------------ | -------- |
| 同级兄弟之间       | `gap-*`  |
| 容器内部留白       | `p-*`    |
| 文本基线微调、归零 | `margin` |

不要用一串 `mb-*` 把整个页面堆出来。

**`space-y-*` / `space-x-*` 的处理**：

Tailwind 的 `space-y-*` / `space-x-*` 通过 `> * + *` 选择器给相邻兄弟加 margin，效果近似 `gap`，但本质是自动化的 margin。在当前项目中：

- 优先使用 `flex flex-col gap-*` 替代 `space-y-*`
- 优先使用 `flex gap-*` 替代 `space-x-*`
- 若容器已经是 `flex` 或 `grid`，**禁止** `space-*`，直接用 `gap-*`
- `space-*` 的唯一合法场景：非 flex/grid 容器内的临时过渡，且应在重复出现后上收为 flex+gap

### 3. 先消除原生 margin 叠加，再用 `gap` 建立节奏

**问题**：`antd` 的 `Typography.Title`、`Typography.Paragraph` 等显示型组件默认自带垂直 margin。直接放进 `flex / grid + gap-*` 的 wrapper 时，实际节奏 = 组件 margin + wrapper gap 的叠加值，不可控。

**当前项目的决策**：

- 外层 wrapper 负责主节奏
- `Typography.*` 等显示型子元素进入 wrapper 节奏编排前，需先把原生垂直 margin 归零
- 标准业务表单中的 `Form.Item` **不默认归零**；其垂直节奏和校验预留空间优先交给 antd 自身管理
- 由于项目禁止给 `antd` 组件本体挂自由 `className`，margin 归零优先通过 `style`、组件 API、共享 wrapper 或受控样式层处理

**工程保障（当前已定结论）**：

| 情况                               | 做法                                                   |
| ---------------------------------- | ------------------------------------------------------ |
| 偶发性单个 `Typography` 归零       | `style={{ margin: 0 }}` 临时可接受                     |
| 同类 reset 重复出现超过 2 处       | 上收为共享 wrapper 或受控样式层，停止手写重复          |
| 全站 `Typography` 默认 margin 处理 | **不做**全局抹平；标准表单等依赖内置节奏的场景不能误伤 |

不要长期依赖开发者在每个 `Typography.Title` / `Typography.Paragraph` 上手写 `style={{ margin: 0 }}`。若同类 reset 重复出现，及时上收；但不要为了省事做粗暴的全局 reset。

### 4. 间距主要落在 wrapper，不落在组件本体

与 [ui-stack-rules.md](../ui-stack-rules.md) 保持一致：

- 外部节奏由 wrapper 控制
- 组件本体优先用自身 API 和它的默认节奏
- 需要控制外部间距时，让调用方自己包 wrapper

**禁止**把 Tailwind 间距类直接打到 `Card`、`Modal`、`Drawer`、`Form.Item`、`Typography.Title` 这类已自带结构语义的组件本体上。

### 5. 只用有限档位

当前项目有效档位（详细数值见 [tokens.md](./tokens.md)）：

- `p-3` / `p-4` / `p-6`
- `gap-1` / `gap-2` / `gap-4` / `gap-6`

补充约束：

- `gap-3`（12px）允许用于行内紧凑对齐（`[图标]+[文字]+[操作]` 同行），也允许用于紧凑垂直分组（标题+说明文案对、简短卡片头），但不用于页面或区块级主节奏
- 方向性 padding（`px-*`、`py-*`、`pt-*`、`pb-*`）同样只使用约定档位值（3/4/6），不要出现 `px-5`、`py-7` 等
- `margin` 若必须使用，优先跟随 4px 网格，不单独发明零碎值
- **禁止**：`p-5`（20px）、`p-7`（28px）、`gap-5`（20px）等非约定档位
- **禁止**：在 antd `Flex` 组件的 `gap` prop 或 CSS `gap` 属性中使用不在 4px 网格上或不在约定档位中的像素值（如 `gap={20}`）

### 6. antd `Flex` 组件的 `gap` prop 遵守同一套档位

antd 的 `<Flex gap={N}>` 接受像素值。该 prop 的值必须落在约定档位上：

| `gap` prop 值 | 等价 Tailwind | 说明         |
| ------------- | ------------- | ------------ |
| `4`           | `gap-1`       | 极小间距     |
| `8`           | `gap-2`       | 行内间距     |
| `12`          | `gap-3`       | 紧凑对齐     |
| `16`          | `gap-4`       | 条目标准间距 |
| `24`          | `gap-6`       | 区块间距     |

不要出现 `gap={20}`、`gap={18}`、`gap={10}` 等非约定值。若从设计上觉得 16 偏紧、24 偏松，优先选择结构调整，而不是发明中间值。

- 页面区块多用 `gap-6`

## 页面上下文

与 [typography.md](./typography.md) 一致，间距节奏也区分两种页面上下文。

### 工作台页面（Workbench）

登录后的主工作区页面，由 `AppLayout` 壳层包裹。

- 壳层已提供外层 padding 和顶部控制栏
- 页面内容区由 `<Flex vertical gap={...}>` 组织主节奏
- 响应式压缩：壳层在窄屏时将主 gap 从 24px 收缩为 16px（`mainWidthBand === 'compact' ? 16 : 24`）
- 页面内部组件遵循标准内容节奏

已落地参考：`src/app/layout/app-layout.tsx` 中的 `Layout.Content` 区域。

### 公共入口页面（Public Entry）

登录前的独立页面（登录、找回密码、验证入口），无 `AppLayout` 壳层，采用左右两栏布局。

间距特点：

| 位置                        | 间距                                               | 说明                                                      |
| --------------------------- | -------------------------------------------------- | --------------------------------------------------------- |
| 页面外壳 padding            | `px-6 py-12`                                       | 全屏页面留白                                              |
| 左右两栏之间                | `Flex gap={32}`                                    | 两栏结构分隔；`wrap` 确保窄屏堆叠时同样保持 32px 垂直间距 |
| 左栏内部（语义标题 + 说明） | `Flex vertical gap={24}`                           | 介绍区段落间距                                            |
| 右栏 Card 内部              | `body: { padding: 24 }` + `Flex vertical gap={24}` | 操作卡片内部                                              |
| 标题与说明文案（紧凑对）    | 自然块级堆叠                                       | 标题 `marginBottom: 12`，说明 `marginBottom: 0`           |

公共入口的 `gap={32}` 是项目唯一允许超出标准内容档位的 layout 例外。这个值用于两栏结构分隔，不可扩散到工作台页面。

已落地参考：`src/pages/login/index.tsx`、`src/pages/forgot-password/index.tsx`、`src/pages/verification-intent/index.tsx`。

## 结构节奏

### 页面级

| 位置               | 间距    |
| ------------------ | ------- |
| 页面头与首个内容区 | `gap-6` |
| 内容区与内容区之间 | `gap-6` |
| 页面内卡片组       | `gap-4` |

不要把页面主节奏收紧到 `gap-4`，除非是明显的紧凑后台视图或小屏单列。

### 内容区级

| 位置                       | 间距    |
| -------------------------- | ------- |
| 区标题与内容主体           | `gap-4` |
| 内容主体（卡片组或列表组） | `gap-4` |
| 内部再分子区块             | `gap-4` |

### 卡片级

| 位置                             | 间距                |
| -------------------------------- | ------------------- |
| 卡片内边距（标准）               | `p-4`               |
| 卡片内边距（信息密度高或阅读型） | `p-6`               |
| 卡片内边距（紧凑）               | `p-3`（限紧凑卡片） |
| 卡片头、正文、底部操作区之间     | `gap-4`             |

`p-3` 只用于明确的紧凑卡片，不要扩散成所有卡片的默认值。

### 表单级

- 标准业务表单：字段垂直节奏优先交给 `Form` / `Form.Item`，不重写
- 表单分组与分组之间：外层 wrapper 用 `gap-6`
- 同一分组内的标准纵向字段流：不再额外用 `gap` 重做每个 `Form.Item` 的主节奏
- 单个 `Form.Item` 内部（标签、输入、帮助文案）：依赖 antd 自身节奏，不拆开用 `gap` 重排

**以下场景才考虑受控重排 `Form.Item`**：

- inline 单字段编辑
- 紧凑面板中的小型表单
- 明确做过错误态验证的自定义表单壳

这些场景下：可用外层 wrapper `gap-4`，可在共享壳层中受控归零 `Form.Item` 的默认 margin；但必须验证错误提示、帮助文案和校验状态不会互相顶压。

### 弹层级（Modal / Drawer）

| 位置               | 间距    |
| ------------------ | ------- |
| 头部与正文         | `gap-4` |
| 正文内主要区块之间 | `gap-4` |
| 正文与底部操作区   | `gap-6` |

短确认型 `Modal` 可以比页面更紧凑，但不要把复杂表单塞进只够 `gap-2` 的窄节奏里。

### 空状态级

| 位置                       | 间距                     |
| -------------------------- | ------------------------ |
| 标题与说明                 | `gap-2`                  |
| 说明与操作                 | `gap-4`                  |
| 整个空状态块与外层容器边界 | 依容器自身 `p-4` / `p-6` |

不要只靠一段灰字孤零零放着，也不要把空状态按钮贴在说明文字下面。

## `gap`、`padding`、`margin` 的快速判断

| 属性      | 适合场景                                                           |
| --------- | ------------------------------------------------------------------ |
| `gap`     | `flex / grid` 同级元素之间；页面区块编排；卡片内部纵向节奏         |
| `padding` | 卡片壳；面板壳；空状态壳；Sidecar / 第三工作区内容壳               |
| `margin`  | 归零组件默认 margin；小块文本微调；无法通过 wrapper 调整的局部例外 |

| ---------------------- | ------------------ |

## Sidecar 与第三工作区

Sidecar 允许比 `main` 更紧凑，但不能乱。

**关键约定**：

- `main` 默认按标准节奏：主区块 `gap-6`
- Sidecar 内部主节奏由 `src/index.css` 中的 `[data-layout-surface='sidecar']` 选择器统一控制，对应以下 CSS 变量：

  | 变量                            | 默认值 | compact 值 | 语义             |
  | ------------------------------- | ------ | ---------- | ---------------- |
  | `--sidecar-root-gap`            | 16px   | 12px       | 根栈主间距       |
  | `--sidecar-stack-gap`           | 16px   | 12px       | 滚动栈子区间距   |
  | `--sidecar-panel-padding`       | 16px   | 12px       | 面板内边距       |
  | `--sidecar-card-padding-block`  | 8px    | 8px        | 入口卡片上下边距 |
  | `--sidecar-card-padding-inline` | 12px   | 10px       | 入口卡片左右边距 |

- **不要**在 Sidecar 内部主要内容栈上再硬写 `gap-*` 类，会覆盖 density 变体
- 局部行内小块仍可用 `gap-2` / `gap-1`；禁止的是把 Sidecar 主内容栈重新写死成 `gap-4` / `gap-6`

**React 侧推荐写法**：

```tsx
<section data-layout-surface="sidecar">
  <div className="sidecar-root-stack">
    <div className="sidecar-scroll-stack">{/* sidecar sections */}</div>
    <div className="sidecar-input-shell">{/* sender / actions */}</div>
  </div>
</section>
```

**第三工作区**：

- 承载长结果物时，回到接近 `main` 的阅读节奏，不要沿用 Sidecar 的紧凑排法
- 壳层优先 `p-6`，主体区块之间优先 `gap-6`
- 若需要比 `p-6` / `gap-6` 更宽松的沉浸式阅读节奏，应先补正式 token，再使用，不要临时发明 `p-8` / `gap-8`

## AI 生成代码时的默认判断

> 本节与 [ai-rules.md](./ai-rules.md) 间距部分保持一致。具体档位约束以 `ai-rules.md` 为准；本节补充的是节奏语义，即"为什么用这个值"。

| 场景                           | 默认值  | 说明               |
| ------------------------------ | ------- | ------------------ |
| 页面级垂直节奏                 | `gap-6` | 区块间主间距       |
| 内容区、卡片组、表单组         | `gap-4` | 条目间标准间距     |
| 行内图标、标签、按钮组         | `gap-2` | 行内紧凑间距       |
| 容器内边距（标准）             | `p-4`   | 大多数卡片壳默认值 |
| 容器内边距（阅读型或重点卡片） | `p-6`   | 阅读密度较低时使用 |

操作顺序：

1. 先用 wrapper `flex flex-col gap-*` 排版，不要用 `space-y-*`
2. 再把 `Typography` 等组件默认 margin 归零（`style={{ marginBottom: 0 }}`）
3. antd `<Flex>` 的 `gap` prop 只传约定档位像素值（4/8/12/16/24）
4. 标准业务表单不默认重写 `Form.Item` 的垂直节奏
5. Sidecar 主栈优先复用 `sidecar-root-stack` / `sidecar-scroll-stack`，不直接写死 `gap-*`

## 禁止项

- 不要把页面主节奏建立在零散 `mb-*` 上
- 不要把 Tailwind 间距类直接打到有结构语义的组件本体上（`Card`、`Modal`、`Drawer`、`Form.Item`、`Typography.Title` 等）
- 不要同时混用 `gap` 和成串 `margin-bottom` 表达同一层级节奏
- 不要把 `gap-3` 当成页面或区块默认间距
- 不要为了"紧凑"把 `p-3` 和 `gap-2` 扩散成全站默认
- 不要使用临时硬编码值，如 `marginTop: 18`、`padding: 20`
- 不要使用 `p-5`、`p-7`、`gap-5` 等非约定档位
- 不要在 antd `Flex` 组件的 `gap` prop 中使用非约定值（如 `gap={20}`、`gap={10}`）
- 不要在 Sidecar 主内容栈上硬写 `gap-4` / `gap-6`，会覆盖 density 变体
- 不要在未补正式 token 的情况下临时发明 `p-8` / `gap-8`
- 不要用 `space-y-*` / `space-x-*` 替代 flex/grid 容器的 `gap-*`；已有 `space-y-*` 应逐步迁移为 `flex flex-col gap-*`
- 不要用 `mb-*` / `mt-*` 在父容器中建立兄弟元素之间的结构节奏（应改用外层 flex/grid + gap）
- 不要在公共入口页面之外使用 `gap={32}` 这类超出标准内容档位的 layout 级间距

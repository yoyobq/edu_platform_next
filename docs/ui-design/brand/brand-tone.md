# Brand Tone

本文件不是硬规则清单，而是品牌相关视觉判断的默认准则。

适用场景：

- AI 生成页面或组件时，不确定品牌存在感该强还是弱
- 设计或实现评审时，判断一个界面是不是“加多了”
- 在 Logo、图标、强调色、水印、留白之间做取舍

若明确需求与本文件冲突，以需求为准；若无明确需求，默认按本文件收口。

## 核心气质

- 克制，但不空
- 规整，但不僵
- 有识别度，但不靠堆砌证明“被设计过”

设计感优先来自：

- 结构清楚
- 比例稳定
- 对齐干净
- 留白成立
- 一两个准确的视觉动作

而不是来自：

- 重复出现的 icon
- 多种装饰语言并列
- 到处都在强调
- 颜色、边框、角标、底纹一起发声

## 适用边界

这套气质不能直接等同于“所有页面都尽量少”。

更适合优先靠近这套气质的场景：

- 公共入口页
- 品牌说明页
- 封面、演示页
- 留白较多的轻信息页面

不应直接照搬“越少越好”的场景：

- 表单密集页
- 表格页
- 监控、状态、权限页
- 调试、代码、工具页
- 多卡片工作台

这些页面首先要保证：

1. 信息能被快速扫读
2. 状态能被立即辨认
3. 操作入口足够明确

在这些前提成立后，才谈克制和减法。

默认优先级：

1. 任务清楚
2. 状态明显
3. 结构稳定
4. 品牌气质

若“更克制”会削弱以上前三项，则不继续减。

## 默认判断

拿不准时，默认按下面的顺序做减法：

1. 先保留结构，不先保留装饰
2. 先保留标题和操作，不先保留图标和点缀
3. 先保留一个记忆点，不并列保留多个记忆点
4. 先保留留白，不先补满空位

简单说：

- 拿不准时，少放一个 icon
- 拿不准时，少加一层装饰
- 拿不准时，少用一种颜色
- 拿不准时，不做水印
- 但不要为了克制而削弱状态提示、操作可见性和信息辨识速度

## 一屏的品牌声量

一个界面通常只需要一个主声部，最多一主一辅。

常见主声部包括：

- 品牌位
- 页面标题
- 主操作
- 状态提醒
- 弱品牌露出

不建议同屏同时把以上内容都做得很响。

建议优先级：

- 正式业务页：标题 / 操作优先，品牌退后
- 公共入口页：品牌 / 标题可以更明确
- 封面、演示页：品牌可以更靠前，但仍不建议同时堆很多功能性装饰

## 图标判断

图标是锚点，不是填空物。

默认做法：

- 先判断它是不是在表达语义，而不是在填空
- 先保留一个清楚的锚点，不并列堆多个差不多有道理的图标
- 若一个标题必须靠 icon 才成立，应先回头检查排版和分组

拿不准时：

- 少放一个 icon
- 先不加第二个 icon

## 颜色判断

品牌色不等于品牌要铺满界面。

默认做法：

- 中性色负责大面积承载
- Deep Blue 负责主交互和少量焦点
- Sky Cyan 负责 AI 提示性存在
- Red 只用于风险和错误

拿不准时：

- 不额外补第三种强调色
- 不把强调色扩成整块大面积背景
- 不让多个强调色同时高频出现

## 水印与弱露出

弱露出可以有，但必须退后。

拿不准时，默认不用水印。

## 评审速查

看到一个界面时，可快速问这几个问题：

1. 这个界面的设计感，主要来自结构还是来自额外装饰？
2. 有没有两个以上元素在抢“记忆点”位置？
3. 图标是在表达语义，还是在填空？
4. 品牌露出是在归属页面，还是在干扰任务？
5. 去掉一半装饰后，这个界面会不会反而更稳？

若第 5 个问题的答案是“会”，那通常就该继续做减法。

# Brand Tone

品牌视觉判断的默认准则。适用于生成页面、评审界面、在品牌元素之间做取舍。

若明确需求与本文件冲突，以需求为准；若无明确需求，默认按本文件收口。

## 品牌定位

面向教育场景的 AI 协作平台。教师管理课程、项目与工作流；学生接收任务、提交成果；AI 在双方需要时介入。

产品身份：**专业工具**。界面第一职责是让任务顺利推进，AI 是协作者，不是视觉主角。

三色 Logo 表达这层关系：

| 角色         | 色值      | 品牌含义                   |
| ------------ | --------- | -------------------------- |
| Deep Blue    | `#1255CC` | 稳定、专业、可信赖         |
| Claude Coral | `#CC6B46` | 辅助、提示、在被需要时出现 |
| Signal Red   | `#D93025` | 只在值得停下来的时刻亮起   |

## 核心气质

三句话：**克制但不空 · 规整但不僵 · 有识别度但不靠堆砌**。

设计感来自结构、比例、对齐、留白、一两个准确的视觉动作。不来自重复的 icon、多种装饰并列、到处强调、颜色边框角标底纹一起发声。

## AI 层与品牌层

Sky Cyan 是 Logo 的一部分（气泡 + i 竖杆），但 UI 中 AI 的视觉声音使用 Claude Coral，Deep Blue 是产品基调。两者协作，不竞争。

- AI 入口和响应区：Claude Coral 用边框、图标色、浅背景轻声存在，不大面积铺写
- 主操作区、表单、表格、卡片：维持 Deep Blue 基调，AI 色退为辅助信号
- Claude Coral 同屏密度过高 → 通常是功能分区问题，不是颜色问题

品牌信号：**正经的专业工具，AI 在你需要时出现**。

## 判断优先级

无论何种页面，始终按此顺序判断：

1. 任务清楚
2. 状态明显
3. 结构稳定
4. 品牌气质

若"更克制"会削弱前三项，不继续减。

## 适用边界

| 优先靠近这套气质         | 不直接照搬"越少越好"               |
| ------------------------ | ---------------------------------- |
| 公共入口页、品牌页、封面 | 表单密集页、表格页、监控 / 权限页  |
| 留白较多的轻信息页面     | 调试 / 代码 / 工具页、多卡片工作台 |

后者首先保证信息可扫读、状态可辨认、操作入口明确，在此基础上才谈克制。

## 品牌声量分配

一屏只允许一个主记忆点，最多一主一辅。

| 页面类型      | 主记忆点           | 辅助               | 禁止                   |
| ------------- | ------------------ | ------------------ | ---------------------- |
| 公共入口页    | Logo 或产品名      | 产品定位说明       | 正文区重复 Logo        |
| 工作台首页    | 主操作区或 AI 入口 | 顶部品牌位（退后） | 品牌位与操作区同级争抢 |
| 内容详情页    | 内容本身           | —                  | 额外品牌装饰           |
| AI 响应区     | Claude Coral 外壳  | —                  | 叠 Logo 或图形装饰     |
| 表格 / 表单页 | 操作入口或状态标识 | —                  | 用品牌色做大面积背景   |

声量优先级：

- 正式业务页 → 标题 / 操作优先，品牌退后
- 公共入口页 → 品牌 / 标题可更明确
- 封面 / 演示页 → 品牌可靠前，但不堆功能性装饰

## 默认减法顺序

拿不准时：

1. 先保留结构，不先保留装饰
2. 先保留标题和操作，不先保留图标和点缀
3. 先保留一个记忆点，不并列保留多个
4. 先保留留白，不先补满空位
5. 默认不用水印

## 颜色声量

- 中性色负责大面积承载
- Deep Blue 负责主交互和少量焦点
- Claude Coral 负责 AI 提示性存在
- Signal Red 仅错误 / 风险

禁止：额外补第三种强调色、把强调色扩成大面积背景、多种强调色同屏高频出现。

详细颜色规则见 [colors.md](../colors.md)。

## 图标声量

- 图标是语义锚点，不是填空物
- 一行若已有 leading icon，谨慎再加第二个
- 标题不应靠 icon 才成立

详细图标规则见 [ui-icons.md](./ui-icons.md)。

## 评审速查

| #   | 检查                     | 若答案为"是" |
| --- | ------------------------ | ------------ |
| 1   | 设计感主要来自额外装饰？ | 回到结构     |
| 2   | 两个以上元素抢记忆点？   | 只留一个     |
| 3   | 图标在填空而非表达语义？ | 移除         |
| 4   | 品牌露出在干扰任务？     | 降级或移除   |
| 5   | 去掉一半装饰后更稳？     | 继续减法     |

# Logo

Logo 用于表达“这是哪个产品”，不用于表达“这里能点”或“这是一个状态”。

## 使用位置

合法场景：

- 顶部品牌位
- 登录前公共入口的说明区
- 对外截图、演示封面、文档封面等品牌归属场景

默认不推荐：

- 页面正文卡片头部重复放 Logo
- 列表项、表格行、Tag、按钮内部放 Logo
- 把 Logo 当作 AI 功能图标或操作 icon

## 页面规则

- 顶部品牌位优先使用“Logo + 产品名”或“文字品牌”二选一
- 若顶部信息密度高，当前阶段可继续保留文字品牌，不强制替换成图形 Logo
- 公共入口页可在左栏展示品牌，不放进右侧操作 Card
- 正式业务页面正文默认不重复展示 Logo

## 弱露出与水印

以下为审美建议，不是强制规则：

- 若页面已经有清楚的顶部品牌位，正文区域一般不再追加第二次品牌露出
- 若确实需要弱露出，优先考虑背景级、低对比、不可交互的存在方式，例如页脚附近或大面积留白区的弱水印
- 水印只能作为氛围层，不参与信息层级竞争；不能影响正文可读性、表单可用性、表格扫描或代码阅读
- 如果弱露出让页面显得更重、更吵或更像“贴了一个 Logo”，宁可不要

## 尺寸与留白

推荐高度：

- 顶部品牌位：`24px`
- 公共入口左栏：`32px`

约束：

- 不要压到小于 `20px`
- 四周保留独立留白，不贴边框、Tag、按钮或分割线
- 和文字品牌并排时，默认使用 `gap-2`
- 默认先追求结构上的规整对齐；若品牌图形本身重心特殊，允许做极小幅度的视觉微调，但只用于修正视觉平衡，不用于“手感式”挪动位置

## 颜色与背景

- 默认使用标准品牌配色版本
- 深色背景优先使用反白版或深底版
- 没有专版时，宁可回退成文字品牌，也不要临时改色
- 禁止改成功能态颜色，如成功绿、告警黄、AI 青色

## 禁止事项

- 禁止拉伸、压扁、旋转、加描边、加阴影特效
- 禁止拆开内部元素重组
- 禁止把 Logo 放进按钮里充当按钮 icon
- 禁止同一屏重复出现多个同级 Logo
